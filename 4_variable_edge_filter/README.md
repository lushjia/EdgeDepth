# Select variable edges
This pipeline selects variable edges from the filtered normalized edge-by-sample matrices generated in Step 3.
Variable edges are edges for which some samples carry an allele state that differs from other samples.
Because edges are represented by normalized edge depth rather than discrete genotypes, this pipeline estimates 
the genotype state of each edge in each sample by fitting a constrained Gaussian mixture model (GMM), same as 
the model used by Genome STRiP. Sample numbers in each genotype state are inferred based on the weight of each 
fitted Gaussian components. Edges for which at least N samples, with default N = 10, carry a genotype state 
different from the majority are selected for downstream analysis.

The pipeline outputs:    
- A varialbe edge list file, with one edge per row.    
- A filterd normalzied edge-by-sample matrix containging only the **variable** edges.

## Pipeline overview
This pipeline performs the following steps:

1. retain edges with depth ≥ min_depth in ≥ min_samples samples.    
2. drop edges whose mean depth across samples exceeds max_mean_depth
3. split each chromosome's edges into chunks, fit the GenomeSTRiP GMM to each chunk in
parallel, then merge all per-chunk pass/fail results into one file per chromosome.
4. re-examine GMM-failed edges with a 4-MAD criterion to rescue genuinely variable edges,
then combine GMM-pass and rescued edges into the final kept edge list for each chromosome.
5. subset each chromosome's normalized edge depth matrix down to only the edges in the final kept edge list.

## How to run the pipeline
```
nextflow run variable_edge_filter.nf -profile mccleary \
    --norm_depth_dir ../3_redundancy_filter/results \
    --scripts_dir    scripts \
    --outdir         results \
```
Add `-resume` to resume from cached work after a failed or interrupted run.

## Required input data
| Parameter | Input type | Description |
|:----|:------|:----------|
| `--norm_depth_dir` | Directory | Directory containing the filtered normalized edge-depth matrices generated in Step 3. |
| `--scripts_dir` | Scripts directory | Directory containing the Python scripts used in this step. |
| `--outdir` | Directory | Directory for output files. |

## Additional editable parameters
| Parameter | Input type | Description |
|:----|:------|:----------|
| `--min_depth` | Number | Minimum depth required for an edge to be considered present in a sample. Used in <br>Step 1. Default: 5. |
| `--min_samples` | Number | Minimum number of samples required to have depth ≥ min_depth for an edge to be <br>retained. Used in Step 1. Default: 10. |
| `--max_mean_depth` | Number | Maximum allowed mean depth across samples. Edges with mean depth greater than <br>this value are removed. Used in Step 2. Default: 1000. |
| `--chunk_size` | Integer | Number of edges per chunk when splitting chromosome edge files for parallel GMM <br>fitting. Used in Step 3. Default: 2000. |
| `--expected_depth` | Number | Expected normalized edge depth for one copy, required by the Genome STRiP GMM. <br>Used in Step 3. Default: 17. |
| `--gmm_min_outlier_samples` | Number | Minimum number of samples required outside the dominant GMM component for an <br>edge to be considered variable. Used in Step 3. <br> Default: 10. |
| `--mad_threshold` | Number | Estimated one-copy depth cutoff, calculated as the mean minus two standard <br>deviations of the one-copy distribution. Used in Step 4. <br> Default: 4.71. |
| `--mad_min_outlier_samples` | Number | Minimum number of samples required outside the median ± 4 × MAD range for a <br>GMM-failed edge to be rescued as variable. Used in Step 4. Default: 10. |

More details about each step and parameter can be found in the Nextflow pipeline.

## Output file
The pipeline produces the following outputs:         
1. A text file containing one variable edge per row (`chr*.variable_edge_list.final.txt`).    
Example:    
```
>105403289>105403291    
>105401910>105401912
```
2. Filtered normalized edge-by-sample matrix (`chr*.variable_edge_list.normalized_depth.txt`).    
A normalized edge-depth matrix filtered to retain only the variable edges identified by this pipeline. 
This file is generated by subsetting the filtered normalized edge-by-sample matrix from Step 3.
